What to Test (2026-06-28)

Build: Trip Rework Date: June 28, 2026
Build Focus: Trip start/destination rework and P0 tracking/geofence fixes.

What changed in this build

  • Removed the "Starting point: Home / Current Location" toggle. A trip now always starts from your current location; Home/Stay/Favorites/Recents are destinations.
  • New one-tap Trip Home and Round trip from here (destination-less, for hikes/walks).
  • P0 fixes: destination-less round trips now track + complete; geofences fire reliably via live GPS.

What to Test

This build reworks how trips start and how arrival is detected. Please test the different trip modes on a real device, with Location set to "Always," and actually move so geofences trigger.

★ Trip modes — the focus of this build ★

Every trip starts from your current location. Test each mode and confirm it auto-completes on arrival:

  1. Trip Home — tap Trip Home. Confirm time + contacts → Start. Head home → it should complete automatically when you arrive.
  2. Round trip from here — tap Round trip from here. Set a "be back by" time + contacts → Start. Leave the area, then return → it should complete when you're back.
  3. One-way to a destination — tap Start a Trip, search a destination, leave it as One Way → Start. Arrive → completes.
  4. Named round trip — Start a Trip, pick a destination, choose Round Trip → Start. Go to the destination, then back to your start → completes.

Onboarding & Safety Path

  • Fresh Install: Test the "Set Up Safety" screen (Location + Contacts).
  • No Contacts: Verify you are prompted to add a contact before starting a trip.
  • Safety Path: Don't check in by the deadline → verify the check-in overlay appears and contacts are alerted if you don't respond.
